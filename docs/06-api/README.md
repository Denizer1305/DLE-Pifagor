# API

## Назначение

Раздел описывает правила работы с API ЦОС "Пифагор": авторизацию, роли, ошибки и документацию схемы.

## Авторизация

API использует JWT access/refresh token. Клиент должен:

- прикладывать access token к защищенным запросам;
- обновлять access token через refresh endpoint;
- перенаправлять пользователя на страницу входа при невозможности обновления;
- не отправлять защищенные запросы без токена.

## Роли

Доступ к личным кабинетам и административным endpoint зависит от роли пользователя и активной роли. Ошибка `401` означает отсутствие действительных учетных данных. Ошибка `403` означает, что пользователь авторизован, но не имеет прав на действие.

## Документация схемы

Если в окружении включен drf-spectacular, используйте Swagger/ReDoc endpoints backend для просмотра актуального API.

## Frontend API-клиенты

Frontend хранит API-клиенты внутри модулей. Низкоуровневые запросы находятся в `api`, сценарии работы с backend - в `services`, преобразования ответа - в `mappers`.
