# Project Structure

## Назначение документа

Этот документ описывает рекомендуемую структуру репозитория ЦОС «Пифагор» после перехода проекта из дипломного MVP в развиваемый продукт.

Проект рекомендуется вести как **монорепозиторий**, в котором рядом хранятся backend, frontend, документация, дизайн-материалы, инфраструктура и вспомогательные скрипты.

---

## Общая структура

```text
pifagor/
 ┣ backend/
 ┣ frontend/
 ┣ docs/
 ┣ design/
 ┣ infra/
 ┣ scripts/
 ┣ docker-compose.yml
 ┣ .env.example
 ┣ README.md
 ┗ CHANGELOG.md
```

---

## Назначение директорий

### `backend/`

Django-приложение, реализующее серверную часть платформы: пользователи, роли, организации, курсы, уроки, задания, журнал, расписание, материалы, уведомления, аналитика и ИИ-помощник.

### `frontend/`

Vue-приложение, реализующее пользовательский интерфейс: публичные страницы, личные кабинеты, ролевые интерфейсы, страницы модулей, дизайн-систему, API-клиенты, stores и маршрутизацию.

### `docs/`

Проектная документация: видение проекта, архитектура, описание модулей, дизайн-система, правила разработки и архитектурные решения.

### `design/`

Дизайн-материалы: логотипы, брендбук, UI-kit, макеты, иллюстрации, презентации и экспортированные ассеты.

### `infra/`

Инфраструктурные файлы: Dockerfile, nginx-конфиги, deploy-инструкции, backup-скрипты, staging/production-настройки.

### `scripts/`

Вспомогательные скрипты: генерация demo data, сброс локальной БД, создание суперпользователя, экспорт схемы, запуск проверок.

## Принцип организации проекта

Структура проекта должна отражать не страницы интерфейса, а предметные области ЦОС: пользователи, организации, обучение, курсы, уроки, задания, журнал, расписание, аналитика, материалы, ИИ и олимпиады.

Так проект сможет развиваться от MVP к большой цифровой образовательной среде без хаотичного роста.
