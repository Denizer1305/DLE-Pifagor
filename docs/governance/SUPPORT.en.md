# Support

## How to get help

Use the internal team channel or GitHub issues if the repository is used publicly for academic review.

## What to include

- short problem description;
- user role and page;
- reproduction steps;
- expected result;
- actual result;
- screenshots or logs without secrets;
- Node, Python, and environment versions for technical issues.

## Responsibility areas

- frontend: pages, components, routes, styles, state, API clients;
- backend: models, permissions, serializers, views, services, selectors, Celery;
- infra: CI, deployment, environment variables, backups;
- docs: README, architecture, user guides.
