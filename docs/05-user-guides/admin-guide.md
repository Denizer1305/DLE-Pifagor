# Инструкция администратора

## Назначение кабинета

Личный кабинет администратора предназначен для управления пользователями, организациями, обращениями, уведомлениями, календарем, заметками и настройками платформы.

## Основные разделы

- Главная - сводка состояния платформы и ближайший план.
- Пользователи - список, фильтры, создание, просмотр и редактирование пользователей.
- Организации - карточки образовательных организаций и административные действия.
- Обращения - заявки пользователей, статусы, вложения и детальный просмотр.
- Календарь - события и план на день.
- Заметки - личные заметки, связанные с датами.
- Уведомления - системные события и пользовательские сигналы.
- Настройки - внешний вид, уведомления, приватность, роли и безопасность.

## Работа с пользователями

Используйте фильтры и таблицу пользователей. Для изменения данных открывайте карточку пользователя или страницу редактирования. Резервная почта, телефон, профиль и роли должны сохраняться через backend API.

## Работа с обращениями

Открывайте детальный просмотр обращения, чтобы увидеть полный текст, контакты, организацию и прикрепленные файлы. После открытия уведомление об обращении может быть отмечено как прочитанное.

## Организации

Модуль организаций хранит данные об образовательных организациях. Учебная структура должна храниться отдельно в модуле структуры.
