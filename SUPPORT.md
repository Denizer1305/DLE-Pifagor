# Поддержка

## Как получить помощь

Для вопросов по проекту используйте внутренний канал команды или GitHub issues, если репозиторий используется публично для учебной проверки.

## Что приложить к обращению

- краткое описание проблемы;
- роль пользователя и страница;
- шаги воспроизведения;
- ожидаемый результат;
- фактический результат;
- скриншоты или логи без секретов;
- версии Node, Python и окружения, если проблема техническая.

## Типовые зоны ответственности

- frontend: страницы, компоненты, маршруты, стили, состояние, API-клиенты;
- backend: модели, permissions, serializers, views, services, selectors, Celery;
- infra: CI, деплой, переменные окружения, backup;
- docs: README, архитектура, пользовательские инструкции.
