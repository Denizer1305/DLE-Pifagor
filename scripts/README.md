# Скрипты

Папка `scripts` содержит проектные утилиты для локальной разработки, обслуживания и подготовки данных DLE-Pifagor.

## Назначение

Скрипты из этой директории автоматизируют повторяемые операции:

- создание суперпользователя;
- генерацию `.env` или вспомогательных конфигураций;
- экспорт схемы API;
- сброс локальной базы данных;
- наполнение демонстрационными данными.

## Структура

- `create_superuser.sh` - заготовка для создания административного пользователя.
- `generate_env.py` - генерация переменных окружения или шаблонов конфигурации.
- `export_schema.py` - экспорт схемы API.
- `reset_local_db.sh` - сброс локальной базы данных.
- `seed_demo_data.py` - загрузка демонстрационных данных.

## Правила добавления скриптов

- Скрипт должен иметь понятное имя по действию, которое он выполняет.
- Опасные операции должны требовать явного подтверждения или быть ограничены локальным окружением.
- Если скрипт зависит от backend или frontend, укажите это в комментарии внутри файла.
- Для Python-скриптов используйте существующее окружение backend, если операция работает с Django или БД.

## Запуск

Перед запуском проверьте, что активировано нужное окружение и заполнены переменные окружения. Для backend-операций обычно требуется перейти в `backend/` и использовать виртуальное окружение проекта.

Примеры конкретных команд стоит хранить рядом со скриптом или в корневом `README.md`, если команда нужна большинству разработчиков.
